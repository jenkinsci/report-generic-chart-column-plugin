Specify additional preset equations in JSON format or provide a URL to a JSON file containing preset equations.

This field accepts either:

  • Single line URL: A URL pointing to a file with definitions, e.g.:
    https://raw.githubusercontent.com/judovana/jenkins-report-generic-chart-column/master/src/main/resources/io/jenkins/plugins/genericchart/presetEquations
  • Direct JSON content: Paste the JSON definition of preset equations directly into this field (multi-line supported)

The JSON format should follow the same structure as the built-in preset equations. These additional equations will be merged with the default preset equations, allowing you to extend or override the built-in definitions.

See examples. Note that id is mandatory. See more info.

Syntax rules:

  • Empty lines are ignored
  • Behind hash (#) are comments and examples
  • First comment is mandatory and serves as the ID
  • After comments follows the expression itself following ParserNG rules
  • Expressions can be multi-lined for readability (as ParserNG's --trim)
  • Parameters are substituted into expressions via /*1*/ for first param, /*2*/ for second, and so on

Example JSON structure: 

{
  "equations": [
    {
      "name": "custom_threshold",
      "description": "Custom threshold check",
      "equation": "L[0] > 100",
      "category": "custom"
    }
  ]
}

Example URLs:

  • https://example.com/preset-equations.json
  • file:///var/jenkins/custom-equations.json

Leave empty to use only the built-in preset equations.

Note: When this field is modified, the preset equations cache will be reset and reloaded.
